# Front_1KR - Практики 13-18

Репозиторий приведён в соответствие с практическими заданиями `№13-№18` по дисциплине «Фронтенд и бэкенд разработка».

Проект представляет собой единое PWA-приложение для заметок и напоминаний, в котором реализованы:

- `Service Worker` и офлайн-работа через кэш
- `Web App Manifest` и установка приложения на устройство
- `App Shell` с динамической загрузкой контента
- `WebSocket` через `Socket.IO` для realtime-уведомлений между вкладками
- `Push API` и `web-push` для системных уведомлений
- напоминания с возможностью `snooze` на 5 минут

## Что закрыто по практикам

### Практика 13. Service Worker

- просмотр и добавление заметок
- хранение данных в `localStorage`
- регистрация `Service Worker`
- кэширование статических ресурсов
- офлайн-работа после первого открытия приложения

### Практика 14. Web App Manifest

- добавлен `manifest.json`
- подключён `link rel="manifest"`
- добавлены `meta`-теги для мобильных платформ
- подготовлены PNG-иконки `32x32`, `128x128`, `192x192`, `512x512`
- `Service Worker` кэширует иконки и манифест

### Практика 15. HTTPS + App Shell

- интерфейс построен по архитектуре `App Shell`
- добавлена отдельная страница `О приложении`
- статический каркас кэшируется по стратегии `Cache First`
- страницы из `/content/*` загружаются по стратегии `Network First`
- сервер поддерживает локальный `HTTPS`, если создать dev-сертификат

### Практика 16. WebSocket + Push

- реализован сервер на `Express + Socket.IO + web-push`
- при добавлении заметки рассылается событие `taskAdded`
- во второй вкладке показывается всплывающее realtime-сообщение
- реализованы маршруты `/subscribe` и `/unsubscribe`
- добавлены кнопки включения и отключения уведомлений

### Практика 17. Детализация Push

- добавлена форма заметки с напоминанием
- заметки в `localStorage` содержат `id` и `reminder`
- сервер планирует напоминания через `setTimeout`
- в `Service Worker` реализована кнопка `Отложить на 5 минут`
- маршрут `/snooze` переносит активное напоминание

### Практика 18. Подготовка к контрольной

- проект протестирован вручную по основным сценариям
- README полностью заполнен
- добавлен отдельный отчётный файл [PRACTICE_13_18_REPORT.md](PRACTICE_13_18_REPORT.md)

## Запуск

### 1. Установка зависимостей

```bash
npm install
```

### 2. Запуск приложения

```bash
npm run dev
```

После запуска доступны адреса:

- `http://localhost:3001` - основная версия приложения
- `https://localhost:3443` - HTTPS-версия, если предварительно создан локальный сертификат

## Как включить HTTPS на Windows

Проект умеет запускать HTTPS-сервер, если в `backend/certs` есть локальный сертификат. Для Windows добавлен скрипт:

```bash
npm run cert:windows
```

Что делает скрипт:

- создаёт self-signed сертификат для `localhost`
- экспортирует его в `backend/certs/localhost.pfx`
- добавляет сертификат в хранилище доверенных корневых сертификатов текущего пользователя

После этого перезапустите сервер и откройте:

```text
https://localhost:3443
```

## Как проверить требования практик

### Практики 13-15

1. Откройте приложение.
2. Добавьте несколько заметок.
3. Откройте DevTools -> `Application`.
4. Проверьте:
   - вкладку `Service Workers`
   - `Manifest`
   - `Cache Storage`
5. Отключите сеть и обновите страницу: интерфейс и заметки должны остаться доступными.

### Практика 16

1. Откройте приложение в двух вкладках.
2. В первой вкладке добавьте обычную заметку.
3. Во второй вкладке должен появиться realtime-toast через `WebSocket`.
4. Нажмите `Включить уведомления`.
5. Добавьте ещё одну заметку и проверьте push-уведомление.

### Практика 17

1. Заполните форму напоминания.
2. Укажите дату/время через 1-2 минуты вперёд.
3. Дождитесь системного push-уведомления.
4. Нажмите `Отложить на 5 минут` и убедитесь, что сервер переносит напоминание.

## Что протестировано

Проверены вручную следующие сценарии:

- запуск сервера `npm run dev`
- загрузка главной страницы по `http://localhost:3001`
- загрузка конфигурации сервера `/api/config`
- сохранение обычной заметки в `localStorage`
- сохранение заметки с `reminder`
- регистрация `Service Worker`
- работа `Manifest`
- загрузка страниц `Главная` и `О приложении`
- realtime-уведомления между вкладками через `Socket.IO`
- подписка и отписка от `Push API`
- сборка не требуется, так как фронтенд статический и раздаётся сервером
- синтаксическая проверка backend через `node --check`

## Ограничения

- заметки хранятся только в `localStorage` браузера и не синхронизируются между устройствами
- push-подписки и таймеры напоминаний хранятся в памяти сервера
- после перезапуска backend активные push-подписки и отложенные напоминания сбрасываются
- для корректной демонстрации push-уведомлений сервер должен оставаться запущенным
